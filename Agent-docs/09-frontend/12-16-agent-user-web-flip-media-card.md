# 12-16 Agent User Web Flip Media Card

版本：v1.8.7  
范围：`Agent-user web` 首页 Agent 卡片交互

## 1. 目标

v1.8.7 将用户端首页 Agent 卡片从单面信息卡重构为双面卡片：

- 正面：媒体展示区，展示照片 / 视频。
- 背面：Agent 基础信息，沿用原有昵称、简介、头像首字符策略。
- 翻转：通过 Y 轴 3D 翻转完成。
- 媒体：当前阶段只做浏览器本地预览，不上传、不持久化。

## 2. 视觉结构

### 2.1 卡片正面

正面占满 3:4 卡片区域：

- 图片使用 `object-fit: cover` 填充卡片。
- 视频使用 `object-fit: cover` 填充卡片。
- 左上角显示媒体指示点：
  - 当前媒体：`#0D21A5` 蓝点。
  - 其他媒体：灰白点。
- 右上角显示透明液态玻璃 `i` 按钮。
- 底部显示透明液态玻璃 `start` 按钮。
- 无媒体时，中间显示 `+` 上传按钮和提示文案。

### 2.2 卡片背面

背面保留原来的信息卡：

- 蓝底白色首字符头像。
- Agent 昵称。
- Agent 简介。
- 底部 `start` 按钮。
- 右上角 `i` 按钮用于翻回媒体正面。

## 3. 交互规则

### 3.1 翻转

- 点击正面右上角 `i`：翻转到背面。
- 点击背面右上角 `i`：翻回正面。
- 切换到其他 Agent 时，默认显示正面。

### 3.2 媒体上传预览

当前无真实后端媒体数据，因此正面中央显示 `+`：

- 点击 `+` 打开浏览器文件选择。
- 支持 `image/*` 和 `video/*`。
- 支持一次选择多个文件。
- 使用 `URL.createObjectURL(file)` 生成本地预览地址。
- 不调用上传接口。
- 不写后端。
- 不写数据库。
- 刷新页面后本地预览消失。

### 3.3 视频播放

- 每次切换到视频媒体时自动播放一次。
- 视频静音自动播放，以符合浏览器自动播放策略。
- 视频播放结束后暂停在尾帧。

### 3.4 卡片切换

继续保留：

- 左滑切换下一个 Agent。
- 右滑切换上一个 Agent。
- 点击左右目标卡片切换。
- 点击底部 dot 切换。

## 4. 不做事项

v1.8.7 不做：

- 后端媒体上传。
- 云存储。
- 用户登录。
- 进入聊天页。
- 后端 Agent 数据读取。
- 真实媒体持久化。

## 5. 后续预留

后续正式接入时，可以把本地预览替换为：

```text
POST /user/media/uploads
GET  /agents
GET  /agents/{slug}
```

但页面组件结构不需要推翻：

- 正面继续消费 `agent.social.galleryImages[]` / `agent.social.galleryVideos[]`。
- 背面继续消费 Agent 基础信息。
- 本地上传逻辑替换为正式 Upload Adapter。
